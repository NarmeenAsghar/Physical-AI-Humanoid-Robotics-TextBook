## üî¥ RAG Chatbot Architecture (Mandatory for Phase 1)

### Component Stack

**Backend (FastAPI)**:
```
api/
‚îú‚îÄ‚îÄ main.py                  # FastAPI app
‚îú‚îÄ‚îÄ routes/
‚îÇ   ‚îî‚îÄ‚îÄ chat.py             # Chat endpoints
‚îú‚îÄ‚îÄ services/
‚îÇ   ‚îú‚îÄ‚îÄ rag.py              # RAG retrieval logic
‚îÇ   ‚îú‚îÄ‚îÄ embeddings.py       # Vectorization (3 lessons only)
‚îÇ   ‚îî‚îÄ‚îÄ agent.py            # OpenAI Agent SDK integration
‚îú‚îÄ‚îÄ models/
‚îÇ   ‚îî‚îÄ‚îÄ schemas.py          # Request/response models
‚îî‚îÄ‚îÄ config/
    ‚îî‚îÄ‚îÄ settings.py         # Qdrant, OpenAI configs
```

**Frontend Integration (Docusaurus)**:
```
docs/src/components/
‚îú‚îÄ‚îÄ ChatWidget.tsx          # Main chat UI
‚îú‚îÄ‚îÄ TextSelectionHandler.tsx # Selection capture (Highlight ‚Üí Ask)
‚îî‚îÄ‚îÄ ChatMessage.tsx         # Message display
```

**Database (Qdrant)**:
```
Collection: textbook_content
Schema:
  - id: string (lesson-section-id)
  - vector: float[] (OpenAI embedding)
  - payload:
      chapter: int (1, 2, or 3)
      lesson: int (1 only for Phase 1)
      section: string
      content: string
      title: string
```

### RAG Workflow

1. **User asks question** (or selects text + asks)
2. **Frontend** sends query + optional selected text to FastAPI
3. **FastAPI** generates query embedding
4. **Qdrant** returns top-k similar content chunks (from 3 lessons)
5. **OpenAI Agent** uses retrieved context + query to generate answer
6. **Response** streamed back to frontend with sources

### Key Features
- ‚úÖ Answer questions about textbook content (3 lessons)
- ‚úÖ Support text-selection-based queries ("What does this mean?")
- ‚úÖ Cite sources (chapter, lesson, section)
- ‚úÖ Stream responses for better UX
- ‚úÖ Handle follow-up questions with context
